Restricting Signups with an Allowlist

 
This feature is only applicable for enterprise installations of CSET.  A user running a local installation on a laptop does not need to register for an account.
 
System administrators can restrict the users who can register for CSET accounts.  This may be useful for an organization that wishes to stand up an instance of CSET on the public internet but wishes to control who can create a user account on their CSET instance.
 
User access can be restricted by adding email patterns to an “allowlist” file on the server.
 
To add email patterns to the allowlist:
·     Locate the ALLOWLIST.txt file in the folder where CSET is installed (e.g., Program Files\DHS\CSET x.x.x.x\Website)
·     Edit the file and add patterns to the file, one per line
For example, to limit signups to users that have an email address in the “xyz.com” domain, add this pattern:
*@xyz.com
 
If no patterns are defined in the allowlist file registration is open to everyone.
Lines beginning with the hashtag/pound sign (#) or two hyphens (--) indicate comments and are ignored by CSET.
